1、 说一下对组件化的理解

  1. 什么是组件：组件的定义和引用&组件封装和组件的复用
  2. 组件的封装：封装视图、数据、变化逻辑
  3. 组件的复用：props 传递、复用

JSX 本质是什么

  1. JSX 语法（标签、JS 表达式、判断、循环、事件绑定）
  2. JSX 是语法糖，需被解析成 JS 才能运行
  3. JSX 是独立的标准，可被其他项目使用

JSX 和 vdom 的关系

  1. 每个组件实例，都有 renderComponent 方法
  2. 为何需要vdom：jsx需要渲染成html，还有rerender
  3. 执行 renderComponent 会重新执行实例的 render
  4. React.createElement和h都生成vnode
  5. render 函数返回 newVnode ，然后拿到 preVnode
  6. 何时patch:reactDom.render(...)和setState
  7. 执行 patch(preVnode, newVnode)
  8. 自定义组件的解析：初始化实例，然后执行render

说一下 setState 的过程

  1. setState 的异步：效果、原因
  2. vue 修改属性也是异步：效果、原因
  3. setState 的过程：最终走到 patch(preVnode, newVnode)
阐述自己对 React 和 vue 的认识

两者的本质区别

  1. 文无第一武无第二，技术选型没有绝对的对与错
  2. 技术选型要考虑的因素非常多
  3. 作为面试者，你要有自己的主见
  4. 和面试官的观点不一致没关系，只要能说出理由

看模板和组件化的区别

  1. vue - 本质是 MVVM 框架，由 MVC 发展而来
  2. React - 本质是前端组件化框架，由后端组件化发展而来
  3. 但这并不妨碍他们两者都能实现相同的功能

模板的区别

  1. vue - 使用模板（最初由 angular 提出）
  2. React - 使用 JSX
  3. 模板语法上，我更加倾向于 JSX
  4. 模板分离上，我更加倾向于 vue
  5. 模板应该和 JS 逻辑分离
  6. 回顾“开放封闭原则”

组件化的区别

  1. React 本身就是组件化，没有组件化就不是 React
  2. vue 也支持组件化，不过是在 MVVM 上的扩展
  3. 查阅 vue 组件化的文档，洋洋洒洒很多（侧面反映）
  4. 对于组件化，我更加倾向于 React ，做的彻底而清晰

两者共同点

  1. 都支持组件化
  2. 都是数据驱动试图

总结问题答案

  1. Vue本质是MVVM框架,由MVC发展而来
  2. React本质是前端组件化框架,由后端组件化发展而来
  3. Vue-使用模板(由angular提出) // React-使用JSX
  4. 模板语法上,我更倾向于JSX // 模板分离上,我更倾向于Vue
  5. 国内使用，首推 vue 。文档更易读、易学、社区够大
  6. 如果团队水平较高，推荐使用 React.组件化和 JSX
