JSX的本质是什么

JSX的语法

  1. html 形式
  2. 引入 JS 变量和表达式
  3. if…else…
  4. 循环
  5. style 和 className
  6. 事件
  • 那么它如何在浏览器运行？
  • JSX 语法根本无法被浏览器所解析

JSX 解析成 JS

  1. JSX 其实是语法糖
  2. 开发环境会将 JSX 编译成 JS 代码
  3. JSX 的写法大大降低了学习成本和编码工作量
  4. 同时，JSX 也会增加 debug 成本

独立的标准

  1. JSX 是 React 引入的，但不是 React 独有的
  2. React 已经将它作为一个独立标准开放，其他项目也可用
  3. React.createElement 是可以自定义修改的
  4. 说明：本身功能已经完备；和其他标准监控和扩展性没问题
  5. 另：有机会录制《1000行代码实现React》，就用 JSX 标准

  1. JSX 语法（标签、JS 表达式、判断、循环、事件绑定）
  2. JSX 是语法糖，需被解析成 JS 才能运行
  3. JSX 是独立的标准，可被其他项目使用
